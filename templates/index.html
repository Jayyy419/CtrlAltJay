A running history of content edits made from this browser — create, edit, delete, reorder, import, and backup actions. It's stored locally, so it won't show actions taken from a different browser or device.

Checks every project and experience's external link and credential URL for a live response, run server-side to avoid browser CORS restrictions.

Rename a skill tag everywhere it's used, or merge it into an existing tag by typing that tag's name as the target.

This exports everything currently in the database as JSON:

  • Projects & experiences (portfolio-items)
  • Resume timeline entries (resume-items)
  • Skills (skills)

Saved to your configured S3 bucket under backups/<timestamp>/. This does not overwrite older backups.

Items deleted from Projects or Experiences land here first, so nothing is lost by accident. Restore them or erase them for good.

      Why These Choices
      
          
      A short tour of the more deliberate decisions behind this stack.
      
          
      
            
      
              
      Server-rendered Flask, not a SPA framework
      
              
      No build step, no bundler, no hydration mismatch — view-source shows exactly what shipped. For a portfolio's scale, a React/Vue toolchain would be solving problems this site doesn't have.
      
            
      
            
      
              
      DynamoDB over a relational database
      
              
      Pay-per-request billing and zero connection-pool management fit a low-traffic personal site better than running (and paying for) an always-on Postgres instance.
      
            
      
            
      
              
      Vanilla JavaScript, not a frontend framework
      
              
      Everything on screen — the tab system, the command palette, the drag-and-drop — is plain DOM manipulation. It keeps the whole client inspectable in one file and forces real discipline about what actually needs to ship.
      
            
      
            
      
              
      Prompt context over a RAG pipeline
      
              
      The Ask AI and Smart Search features pass the site's real content straight into the prompt instead of standing up a vector database. The whole corpus is small enough to fit in context, so embeddings and retrieval infra would be over-engineering for the data volume.
